# Асинхронный HTTP-сервер  

## Внимание!  
Проверка производилась на системе WSL2 с установленным Ununtu 24.04 LTS с одним процессором и двумя потоками.  




Создано два http сервера:  
1. Простой (основной) - для вывода каталога с файлами и возможности их просмотра и/или скачивания.  
2. Тестовый (дополнительный) - для проверки RPS  


## Основной http сервер (server.c)  

Запускается с параметрами по умолчанию:  
- каталог - ./www  
- IP адрес - 127.0.0.1  
- порт - 8080  

1. Для сборки сервера выполнить:  
```sh
make
```

2. Предустановка сервера:  
```sh
make setup
```

3. Запуск сервера (можно запускать без первых 2-х пунктов):  
```sh
make run
```

Для проверки сервера из другой консоли выполнить:  
```sh
curl http://127.0.0.1:8080
```

Запустить браузер с адресом http://127.0.0.1:8080.  
Открыть каталог files. Выбрать любой из файлов. Файл должен загрузиться.  

4. Остановка сервера:  
```sh
make stop
```

5. Очистка сборки:  
```sh
make clean
```


## Дополнительный http сервер (rps_server.c)  

Для проверки сервера необходимо выполнить следующие комнды:  

1. Для сборки сервера:  
```sh
make -f RPS_Makefile
```

2. Для подготовки запуска:  
```sh
make -f RPS_Makefile setup
```

3. Для запуска (можно запускать сразу без предыдущих 2-х запусков make -f ...):  
```sh
make -f RPS_Makefile run
```

4. Для тестирования:  
```sh
make -f RPS_Makefile test
```
Вывод (для одного процессора с двумя потоками):  

```sh
📊 Запуск теста производительности...
wrk -t1 -c100 -d10s --timeout 5s -H "Connection: keep-alive" http://127.0.0.1:8080/test.txt
Running 10s test @ http://127.0.0.1:8080/test.txt
  1 threads and 100 connections
  Thread Stats   Avg      Stdev     Max   +/- Stdev
    Latency     2.36ms    2.54ms  31.23ms   88.70%
    Req/Sec    29.96k    10.12k   48.05k    57.58%
  297428 requests in 10.02s, 28.93MB read
Requests/sec:  29696.97
Transfer/sec:      2.89MB
```
Тест производится на простом текстовом файле. Подробности см. в RPS_Makefile стр.30 раздела setup.  

5. Для остановки сервера:  
```sh
make -f RPS_Makefile stop
```

6. Для очистки проекта:  
```sh
make -f RPS_Makefile clean
```

По умолчанию сервер использует следующие параметры:  

- каталог - ./www  
- IP адрес - 127.0.0.1  
- порт - 8080  

Для запуска сервера с другими параметрами необходимо их изменить следующим образом (можно использовать один из):  
```sh
make -f RPS_Makefile run PORT=12345 IP=192.168.100.100 WWW_DIR=./asdfg
```
