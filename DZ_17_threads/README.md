# Задание  

Написать программу, в заданное число потоков разбирающую логи веб-сервера в стандартном (“combined”) формате и
подсчитывающую агрегированную статистику: общее количество отданных байт, 10 самых “тяжёлых” по суммарному трафику
URL’ов и 10 наиболее часто встречающихся Referer’ов.

★★★☆☆

Получить опыт создания многопоточных приложений.

2. Приложение корректно подсчитывает требуемые статистические данные (файлы логов для проверки прилагаются).  
3. Приложение корректно обрабатывает случай пустой директории с логами.

# Multi-threaded HTTP Log Parser  

Консольное приложение на языке C для высокопроизводительного анализа логов веб-сервера в формате Combined. 
Программа обрабатывает директорию с логами в несколько потоков, подсчитывает общий трафик и формирует топ URL и рефереров.  

## Особенности  
**Многопоточность:** Распараллеливание обработки файлов с использованием pthread.  
**URL Decoding:** Автоматическое преобразование кириллицы и спецсимволов из %XX формата в читаемый текст (UTF-8).  
**Эффективность:** Использование кастомной Hash-таблицы для быстрого подсчета статистики.  
**Масштабируемость:** Автоматическое определение количества ядер процессора для оптимальной производительности.  

## Возможности  
* Многопоточность: Эффективное распределение файлов из директории между заданным количеством потоков (POSIX Threads).  
* Статистика:  
* Общий объем переданного трафика в байтах.  
   * Топ-10 URL-адресов по суммарному количеству переданных байт.  
   * Топ-10 наиболее часто встречающихся Referer (по количеству запросов).  
* Безопасность: Корректная обработка пустых директорий, ошибок доступа к файлам и защита общих данных мьютексами.  
* Отсутствие утечек: Память полностью освобождается (проверено через Valgrind).  

## Сборка через **make**  
Программа написана на стандарте C11 и требует библиотеку pthread.  
```sh
make
```

## Запуск  
По умолчанию программа ищет логи в папке logs и использует количество потоков, равное числу ядер CPU:  
```sh
make run
```

Можно вручную указать путь и количество потоков:  
```sh
make run DIR=/path/to/your/logs THREADS=4
```

## Очистка  
```sh
make clean
```

## Ручная сборка:  
```sh
gcc -std=c11 -Wall -Wextra -Wpedantic parser_logs.c -o parser_logs -lpthread
```

## Использование  
Предварительно создать каталог logs и распаковать туда файл logs.7z.  
Запуск программы осуществляется передачей пути к папке с логами и количества желаемых потоков:  
```sh
./parser_logs <путь_к_директории> <количество_потоков>
```

Пример:  
```sh
./parser_logs ./logs 4
```

## Технические детали  

* Структура данных: Для хранения статистики используются хеш-таблицы с методом цепочек (размер таблицы 1,048,576 бакетов 
для минимизации коллизий на больших данных).  
* Балансировка: Файлы распределяются между рабочими потоками по алгоритму Round-robin.  
* Парсинг: Используется потокобезопасная логика и ручной разбор смещений для точного извлечения байт и рефереров.
