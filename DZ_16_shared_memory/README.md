Написать программу, подсчитывающую контрольную сумму CRC32 (не Adler32) для файлов, превышающих
по объёму оперативную память без многократного вызова read. Сторонние библиотеки не использовать.

# CRC32 File Utility  

Минималистичная и быстрая утилита на языке Си для вычисления контрольной суммы CRC32 (стандарт IEEE 802.3).  

## Особенности  
- **Низкое потребление памяти**: Файл обрабатывается блоками (64 КБ), что позволяет вычислять хеш для файлов, 
объем которых значительно превышает размер оперативной памяти.  
- **Высокая скорость**: Использование табличного метода вычисления вместо побитового.  
- **Никаких зависимостей**: Используются только стандартные библиотеки Си.  
- **Совместимость**: Вывод полностью совпадает со стандартной системной утилитой `crc32`.  

## Сборка и запуск через `make`  
Для сборки проекта применяется компилятор `gcc` и утилита `make`.  
1. Сборка проекта:  
```sh
$ make
```

2. Запуск теста:  
```sh
$ make test
./crc32_tool file.bin

deadbeef
```

3. Запуск теста с изменённым параметром:  
```sh
$ make test FILE=test_clean.txt
./crc32_tool test_clean.txt

e271e75a
```

По умолчанию, используется файл большого размера `file.bin` (68719476736 байт), заархивированный в file.7z.  
Запуск `make test` возможен без предварительного запуска сборки через `make`.  

4. Очистка объектных файлов после сборки:  
```sh 
$ make clean
```

## Использование в командной строке  
Запуск программы с передачей пути к файлу в качестве аргумента:  
```sh
$ ./crc32_tool <путь_к_файлу>
```

## Пример:  
```sh
$ echo -n "Hello Linux" > test_clean.txt
$ ./crc32_tool test_clean.txt
e271e75a
```

Для проверки правильности работы можно использовать системную утилиту crc32 или python запрос:  
1. В WSL2 (WSL2 запускает crc32 через Windows, поэтому результат получается другой):  
```sh
$ perl $(which crc32) test_clean.txt
e271e75a
```

2. В оригинальном Ubuntu:  
```sh
$ crc32 test_clean.txt
e271e75a
```

3. Python запрос:  
```sh
$ python3 -c "import binascii; print(hex(binascii.crc32(open('test_clean.txt', 'rb').read()) & 0xffffffff))"
0xe271e75a
```

## Технические детали  

* Полином: 0xEDB88320  
* Начальное значение (Init): 0xFFFFFFFF  
* Финальный XOR: 0xFFFFFFFF  
* Режим чтения: Бинарный (rb), что гарантирует корректный подсчет на любых типах данных.  

## Обработка ошибок  
Программа корректно обрабатывает:  

* Отсутствие аргументов командной строки.  
* Отсутствие доступа к файлу или его отсутствие (вывод ошибки через perror).  
* Ошибки чтения в процессе обработки данных.
