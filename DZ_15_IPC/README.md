# File Size Daemon (FSD)  

### Минималистичный системный демон на языке Си для мониторинга размера файлов через UNIX-сокеты.  

## Особенности  

* Режимы работы: Поддерживает запуск как в консоли, так и в фоне (режим демона).  
* Hot Reload: Перечитывает конфигурацию и пересоздает сокет «на лету» по сигналу SIGHUP.  
* Динамичность: Обновляет данные о размере файла через stat() при каждом новом клиентском подключении.  
* Безопасность: Корректно обрабатывает ошибки доступа к файлам и ошибки конфигурации.  

## Требования  

* GCC (поддержка стандарта C11)  
* POSIX-совместимая ОС (Linux, BSD)  
* Утилита make для сборки  

## Быстрый старт  

1. Сборка проекта:  
```sh
make
```

2. Создание конфигурации по умолчанию:  
```sh
make config
```

Это создаст файл daemon.conf с путями /tmp/target_file.txt и /tmp/file_size_daemon.sock и 
файл /tmp/target_file.txt.

3. Запуск демона:  
```sh
make daemon
```

4. Проверка (запрос размера):  
```sh
make test
```

5. Запуск приложения  без демонизации:
```sh
make run
```


## Управление  

|   Команда   |                        Описание                     |  
|-------------|-----------------------------------------------------|  
| make        | Компиляция исполняемого файла fsd.                  |  
| ./fsd       | Запуск процесса в интерактивном режиме.             |  
| ./fsd -d    | Запуск в режиме демона (фоновый процесс).           |  
| make reload | Отправка сигнала SIGHUP (перезагрузка daemon.conf). |  
| make clean  | Удаление бинарных файлов и временных объектов.      |  

## Конфигурация (daemon.conf)  
Файл конфигурации должен находиться в той же папке, что и исполняемый файл.  

### Путь к файлу для мониторинга  
```sh
FILE_PATH=/tmp/my_log.log
```

### Путь к UNIX-сокету  
```sh
SOCKET_PATH=/tmp/fsd.sock
```

## Ручное управлене  

## Запуск в режиме демона  
```sh
./fsd -d
```

## Изменение файла /tmp/target_file.txt  
```sh
echo "zxcvbnmqwertyu" >> /tmp/target_file.txt
```

## Остановка:  
```sh
kill $(cat /tmp/file_size_daemon.pid)
```

### Перезагрузка конфига:  
```sh
kill -HUP $(cat /tmp/file_size_daemon.pid)
```

### Детали реализации  

* Сигналы: Использование sigaction вместо устаревшего signal для надежной обработки SIGHUP без прерывания критических операций.  
* Сборка: Компиляция с флагами -Wall -Wextra -pedantic гарантирует чистоту кода и отсутствие утечек логики.  
* Смена сокета: Если в конфиге изменен SOCKET_PATH, демон автоматически закрывает старый сокет, удаляет его файл и биндится к новому пути.
