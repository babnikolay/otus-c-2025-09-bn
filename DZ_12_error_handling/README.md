# Simple C Logger Library  

Легковесная и потокобезопасная статическая библиотека на Си (C11) для журналирования событий. 
Поддерживает различные уровни важности, автоматический вывод места вызова (файл и строка) и 
вывод стека вызовов (backtrace) для ошибок.  

## Особенности  

- **4 уровня логирования**: `DEBUG`, `INFO`, `WARNING`, `ERROR`.  
- **Thread-safe**: Использование `pthread_mutex` гарантирует целостность логов при работе из разных потоков.  
- **Stack Trace**: Автоматический вывод стека вызовов для уровня `LOG_ERROR` (требует `-rdynamic`).  
- **Безопасность**: Обработка ошибок выделения памяти (`malloc`) и некорректных путей к файлам.  
- **Удобство**: Макрос `LOG()` автоматически подставляет имя файла и номер строки.  

## Структура проекта  

- `src/` — исходный код библиотеки (`logger.c`) и тестовое приложение (`test.c`).  
- `include/` — заголовочный файл (`logger.h`).  
- `dist/bin/` — директория для скомпилированного исполняемого файла.  
- `liblogger.a` — собранная статическая библиотека.  

## Сборка и запуск  

Проект собирается с помощью `make`. Используются флаги `-std=c11`, `-Wall`, `-Wextra`, `-Wpedantic`.  

### Требования  
- GCC  
- Библиотека `glibc` (для `execinfo.h` / backtrace)  
- POSIX-совместимая ОС (Linux)  

### Команды  
1. **Сборка всего проекта** (библиотека + тест):  
```bash
make
```

2. **Запуск тестового приложения:**  
```bash
make run
```

3. **Очистка временных файлов:**  
```bash
make clean
```

**Примеры вызовов**  
```bash
LOG(LOG_INFO, "Приложение запущено");
LOG(LOG_DEBUG, "Значение переменной x = %d", x);
LOG(LOG_WARNING, "Внимание: ресурс почти исчерпан");
LOG(LOG_ERROR, "Критический сбой!"); // Выведет стек вызовов в лог
```
